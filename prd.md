# Product Requirements Document (PRD)

## 1. Project Title
**MediVita Hospital Management System (HMS)**

## 2. Problem Statement
Managing hospital operations manually—such as patient records, appointment scheduling, and doctor availability—is inefficient, time-consuming, and prone to errors. This system will provide an automated, digital hospital management solution to streamline healthcare administration and improve the experience for both patients and staff.

## 3. Objectives
- Digitize and automate appointment scheduling and patient records management.
- Provide specialized, role-based portals for Patients, Doctors, and Administrators.
- Ensure secure user authentication and seamless onboarding using both standard login and Google OAuth.
- Centralize data to improve the operational efficiency of the hospital.

## 4. Key Features
- **User Authentication:** Secure, role-based login system (Standard and Google OAuth) for patients, doctors, and admins.
- **Patient Portal:** Allows patients to view their dashboard, book and manage appointments, and access their medical records securely.
- **Doctor Portal:** Enables doctors to view their personalized dashboard, manage their schedules, and check patient details efficiently.
- **Admin Management:** Comprehensive admin panel to oversee all system operations, manage doctor/patient accounts, and monitor overall hospital metrics.
- **Dashboards & Reporting:** Interactive and informative dashboards for all user roles.

## 5. Technologies Used
- **Frontend:** HTML, CSS, Vanilla JavaScript
- **Backend:** PHP
- **Database:** MongoDB (Local Database / MongoDB Atlas)
